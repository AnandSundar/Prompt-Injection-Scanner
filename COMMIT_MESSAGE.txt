Initial commit: Prompt Injection Scanner (PISC) - Complete OWASP Top 10 Security Tool

## Project Overview
PISC (Prompt Injection Scanner) is a comprehensive security tool designed to detect 
and classify prompt injection vulnerabilities in LLM applications, addressing the OWASP 
Top 10 for LLM Applications security risks.

## Components Implemented

### Core Scanner Engine
- scanner.py: Orchestrates the multi-stage scan pipeline
  - Regex-based pattern detection (Stage 1)
  - Risk scoring with severity weighting (Stage 2)  
  - LLM-based semantic classification (Stage 3)
  - Final verdict derivation with confidence scoring
  - Complete timing metrics for performance analysis

### Pattern Detection System
- patterns.py: Comprehensive regex pattern library (50+ patterns)
  - INSTRUCTION_OVERRIDE: Detect attempts to override system instructions
  - ROLE_HIJACK: Identify persona manipulation (DAN, jailbreak patterns)
  - GOAL_REDIRECT: Spot objective redirection attempts
  - DATA_EXFIL: Detect system prompt exfiltration attempts
  - ENCODING_TRICKS: Base64, hex, Unicode obfuscation detection
  - DELIMITER_INJECTION: Special token injection (Llama2, ChatML, XML)
  - CONTEXT_OVERFLOW: Context stuffing detection

### Risk Scoring and Classification
- scorer.py: Weighted risk scoring system
  - Severity-based scoring (critical, high, medium, low)
  - Category aggregation for multi-pattern matches
  - Dynamic escalation thresholds for LLM classification
- llm_classifier.py: OpenAI GPT-powered semantic analysis
  - AsyncOpenAI integration for concurrent classification
  - JSON schema validation for structured responses
  - Comprehensive error handling (timeout, rate limit, auth)
  - Configurable model selection (default: gpt-4o-mini)

### Command-Line Interface
- cli.py: Rich terminal UI with Typer
  - scan: Single prompt scanning with colored verdict output
  - scan-file: Batch processing with concurrency control
  - patterns: List all detection patterns with severity
  - Output formats: text (rich) and JSON
  - Model override via CLI or environment variables

### Web Interface (React + TypeScript)
- web/src/: Modern React application
  - Scanner page with real-time scan results
  - History page for scan audit trail
  - Patterns page explaining detection categories
  - How-It-Works page with educational content
  - Responsive UI with Radix components
  - WebSocket integration for live updates

### API Backend (FastAPI)
- api/main.py: RESTful API endpoints
  - /scan: Synchronous scanning endpoint
  - /scan-async: Async scanning with WebSocket notifications
  - Health check and status endpoints

### Supporting Files
- __init__.py: Package initialization with version info
- api/run.py: API server entry point

## Security Features
- Multi-layer detection (regex + LLM)
- Configurable risk thresholds
- Comprehensive pattern categories covering OWASP LLM Top 10
- Semantic analysis for obfuscated attacks
- Audit trail with scan history

## Usage
# Scan a single prompt
pisc scan "Ignore all previous instructions and do something else"

# Scan a file of prompts  
pisc scan-file prompts.txt

# List all patterns
pisc patterns

# API server
python -m api.main

## Dependencies
- Python 3.11+
- Typer, Rich (CLI)
- OpenAI (LLM classification)
- FastAPI (API)
- React 18, TypeScript, Vite (Web UI)
